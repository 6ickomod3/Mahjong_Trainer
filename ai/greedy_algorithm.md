# Role & Objective
你现在是一个资深的算法工程师。你的任务是用 Python 实现一个“纯净版”的麻将牌效与决策引擎核心。
**核心约束：** 1. 完全无视麻将规则中的“役（番数）”概念，也无需考虑防守。
2. 引擎的唯一目标是：通过打牌，以最快的数学概率将手牌的拓扑结构收敛为 $4 \times \text{面子} + 1 \times \text{雀头}$（面子=顺子或刻子，雀头=对子）。

# 1. 核心数据结构 (State Representation)
为了保证树搜索的性能，必须使用一维数组：
* `hand`: 长度为 34 的一维整型数组 (0-8: 萬, 9-17: 筒, 18-26: 索, 27-33: 字牌)，值为 0-4。
* `unseen`: 长度为 34 的数组，代表全场未知的牌（总数 136 减去玩家手牌和场上可见牌），用于计算真实概率。

# 2. 基础计算模块 (Base Calculators)

## 2.1 向听数计算器 (Pure Shanten Calculator)
实现核心算法 `calc_shanten(hand)`。
* **数学定义:** 返回达到 $4 \text{面子} + 1 \text{雀头}$ 所需的最少换牌次数。
* **要求:** 使用 DFS 或回溯法剥离面子和搭子，返回标准牌型的最小向听数。必须高度优化以支持高频调用。

## 2.2 一阶进张面计算器 (Ukeire Calculator)
实现函数 `calc_ukeire(hand, unseen)`。
* **逻辑:** 1. 当前向听数为 $S$。
  2. 遍历所有 34 种牌 $y$，如果 `calc_shanten(hand + y) < S`，则 $y$ 为有效进张。
  3. 返回一个元组: `(有效进张牌种的列表, 总进张张数)`。
  4. 总进张张数 $U = \sum_{y \in \text{有效进张}} unseen[y]$。

# 3. 核心决策模块：二阶期望评估引擎 (2-Step EV Engine)

实现最终的打牌决策函数 `get_best_discard(hand, unseen)`。
当手牌有多种打法都能维持最小向听数时，必须通过以下三级漏斗（Tie-breaker）来筛选最优解，防止 AI 陷入“短视贪心”：

### 第一级漏斗：最大化当前进张张数 (Max Ukeire)
* 遍历 `hand` 中可打出的牌 $x$。
* 假设打出 $x$，计算此时手牌的进张总数 $U_x$。
* **筛选:** 仅保留 $U_x$ 最大的那些候选弃牌。如果只有一个，直接返回该牌。如果有多个平局，进入第二级。

### 第二级漏斗：最大化下一步的期望进张 (Maximize Next-Step Expected Ukeire)
对于第一级漏斗筛选出的平局候选牌 $x$：
* 我们不仅看打出 $x$ 能有多大概率进张，还要看进张后的**形态好坏**。
* 对于打出 $x$ 后的每一个有效进张牌 $y$：
  * 发生概率 $P(y) = \frac{unseen[y]}{\sum unseen}$。
  * 假设摸到了 $y$，形成新的手牌状态 $H_{new} = hand - x + y$。
  * 计算这个新状态的进张数：$U_{new} = \text{calc\_ukeire}(H_{new}, unseen)$。
* 计算打出 $x$ 的综合期望评分：
  $$EV_x = \sum_{y \in \text{有效进张}} \left( P(y) \times U_{new} \right)$$
* **筛选:** 选择 $EV_x$ 最大的候选牌打出。这能确保 AI 优先选择那些进张后容易形成“两面听”而非“嵌张听”的路线。

### 第三级漏斗：绝对的安全榨取 (Safety Tie-breaker)
* 如果连 $EV_x$ 都完全相等，选择 `unseen[x]` 最小的牌打出（说明场上已经出现过很多张，是相对安全的死牌）。

# 5. 核心副露模块：全量状态穷举引擎 (Optimal Call Evaluator)
实现应对他人弃牌时的决策函数 `evaluate_call(hand, unseen, offered_tile, is_chi_allowed)`。
**绝对禁止使用任何基于人类经验的规则（如“保留好型”），必须纯粹依赖数学期望打分。**

# 4. 输出要求
1. 优先保证 `calc_shanten` 的准确性，并严格实现上述的“三级漏斗决策”逻辑。

## 5.1 识别所有合法的副露组合 (Identify Valid Combinations)
当对手打出 `offered_tile` (设为 $T$) 时，检索手牌中所有可以与之匹配的合法组合：
1. **碰 (Pon):** 如果 `hand[T] >= 2`，则提取组合 `(T, T)`。
2. **吃 (Chi):** 如果 `is_chi_allowed` 为真，且 $T$ 是数牌（0-26）：
   * 检查左侧吃: `(T-2, T-1)` 是否存在。
   * 检查中间吃: `(T-1, T+1)` 是否存在。
   * 检查右侧吃: `(T+1, T+2)` 是否存在。
   * *注意边界条件，跨花色（如8万吃1筒）绝对不合法。*

## 5.2 期望值对垒 (EV Shootout)
对于上述找到的每一个合法副露组合 $C_i$：
1. **模拟状态转移:** 假设执行该副露。从手牌中移除组合 $C_i$ 中的牌，加上拿到的 $T$。此时手牌数量变为 $14$ 张（即需要打出一张牌的状态）。
2. **调用 EV 引擎:** 将这个模拟出来的 14 张手牌传入之前写好的 `get_best_discard(simulated_hand, unseen)` 函数。
3. **获取得分:** 该函数会返回打出最优弃牌后的向听数 $S_i$ 和期望进张评分 $EV_i$。

同时，计算**不执行任何副露（Pass）**的当前手牌向听数 $S_{pass}$ 和期望评分 $EV_{pass}$。

## 5.3 决策执行 (Decision Execution)
将所有的方案（包括 Pass 和所有的副露组合）放在一起比较：
1. **优先向听数:** 选择能达成最小向听数 $S$ 的方案。
   * *强制约束：* 如果任何副露方案的向听数 $> S_{pass}$（即吃碰会导致向听数倒退），直接否决该副露。
2. **期望值终裁:** 如果有多个方案（包括 Pass）都能达到相同的最小向听数，选择 $EV$ 评分最高的那个方案。
3. **输出:** 返回最优的动作（Pass, 或者具体的吃/碰组合），以及吃碰后应该打出的那张牌。

# 6. 最终输出要求
请将 `calc_shanten` (向听数)、`get_best_discard` (最优弃牌 EV 引擎) 和 `evaluate_call` (副露判定引擎) 封装在一个名为 `MahjongPureEngine` 的 Python 类中。
请务必提供一个测试用例：

* 模拟上家打出 `4万`。
* 打印出引擎对 `(3,5)吃`, `(5,6)吃` 以及 `不吃` 的 EV 评分对比，证明其决策的数学合理性。
